---

1. Input files required

| File | Description |
|------|-------------|
| .xpm | Hydrogen bond existence map generated by `gmx hbond-legacy` (with `-hbm`). Contains a matrix: rows = individual H-bonds, columns = simulation frames; symbols `o` (present), `-` (not present), `*` (ambiguous). |
| .log | Output of `gmx hbond` (or `gmx hbond-legacy`) containing the donor–hydrogen–acceptor atom triplets per bond (one line per bond, columns: donor, hydrogen, acceptor). |
| .ndx | GROMACS index file with at least one section containing three numbers per H-bond (donor, hydrogen, acceptor) in the same order as the `.xpm` rows. Typically named `[ hbonds_... ]`. |
| .pdb | Structure file (any frame from the trajectory) used to map atom serial numbers to residue numbers and names. Must contain all atoms referenced in the `.ndx` triples. |

---

2. Outputs generated

Occupancy table: For each individual bond: count of `o`, percentage, number of `-` and `*` symbols.
Correlation matrix: Pearson correlation between all bond time series (heatmap with zoom and high‑DPI save).
Network graph: Nodes = residues; edges = H‑bonds weighted by occupancy. Computed centralities: degree, betweenness, closeness, eigenvector.
Residue occupancy table: For each residue, percentage of frames in which at least one of its bonds is `o` (i.e., the residue is “active”), plus total number of bonds it participates in.
Highlights: Interactive graph with zoom/pan and ability to highlight a specific residue by name (e.g., `ASP114`).

---
3. Step‑by‑step usage

	1. Prepare your GROMACS data  
   Run `gmx hbond-legacy` with options `-hbm hbmap.xpm -hbn hbndx.ndx -num hbnum.xvg` (you may need to redirect output to get a `.log` file manually, or use the terminal log). Ensure the `.ndx` file contains sections with triplets (donor–H–acceptor) for the bond types you want (e.g., protein–ligand, protein–protein, etc.).

	2. Launch the GUI 
   Execute `python3 HBanalysis.py`. A window titled *HBfind - GUI* will appear.

	3. Load input files 
   Use the “Search” buttons to select:  
   - .xpm file (existence map)  
   - .log file (bond triplet list)  
   - .ndx file (index with triplets)  
   - .pdb file (structure for residue mapping)  
   Specify an output `.txt` file path to save the occupancy table.

	4. Compute basic occupancy  
   Click “▶ Calculate”. This reads the `.xpm` and `.log`, computes per‑bond prevalences, and displays the table in the main text area. The results are also saved to the chosen `.txt` file.

	5. Correlation analysis (optional)  
   Click “📊 Correlation”. A new window opens showing the Pearson correlation heatmap between all bonds. You can zoom/pan using the toolbar and save the image as a high‑resolution JPG/PNG.

	6. Network analysis 
   Click “🌐 Nettwork Analysis”. This builds the residue‑based network, calculates centrality metrics, and computes the residue‑wise occupancy table. Results are appended to the main text area. A graphical window opens showing the network:
   - Node colour = degree centrality (viridis scale).  
   - Node size = betweenness centrality.  
   - Edge thickness = occupancy weight.

	7. Interact with the network 
   - Use the matplotlib toolbar (zoom/pan) to explore the graph.  
   - Type a residue name (e.g., ASP114) in the “Highlight residue” box and click “Highlight” to highlight that node in red and enlarge it.  
   - Click “Save as JPG (1200 dpi)” to save the current view at publication quality.  
   - Close the window when finished.

4. Interpreting the results

| Metric                 | Meaning |
|------------------------|---------|
| Bond occupancy (%)     | Thermodynamic stability of each individual H‑bond. |
| Residue occupancy (%)  | Fraction of simulation time the residue is engaged in at least one H‑bond. High values indicate “hotspots” – residues that sustain the interaction network. |
| Degree centrality      | Number of direct H‑bond partners. High degree = central hub. |
| Betweenness centrality | How often a residue lies on shortest paths between others. High betweenness = gatekeeper / allosteric communicator. |
| Closeness centrality   | Average distance to all other nodes. High closeness = efficient signal spreader. |
| Eigenvector centrality | Influence of a node considering the importance of its neighbours. High eigenvector = strategically connected to other important residues. |
| Correlation matrix     | Synchronised behaviour between bonds; clusters of positively correlated bonds may indicate cooperative motions. |

---

5. Typical applications

- Receptor–ligand: Identify anchoring residues (high residue occupancy) and allosteric pathways (high betweenness).  
- Protein–protein interfaces: Map hot‑spot residues and communication between subunits.  
- Protein–DNA/RNA: Locate specificity‑determining contacts.  
- Intra‑protein dynamics: Detect long‑range communication between functional domains.
